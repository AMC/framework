.htaccess
  requires rewrite module
  uses regular expression
  *** use urls as namespacing
    ex) <user>/<repo>/<feature> instead of <feature>/<user>/<repo>
  underscores are bad, use dashes
  design urls for humans
  index file loads by default, cannot use interface.php as launch point
  use REST methods instead of Browser?
   YES! because no conflict over GET anymore!
    Create  / POST
    Read    / GET
    Update  / PUTS
    Delete  / DELETE
  
  rule syntax
    RewriteRule <pattern> <substitution> [flags]
    RewriteCond <test> <condition>
    RewriteMap <map> <path>
   
  flags
    forbidden   403 forbidden
    gone        410 gone
    nocase      pattern is case-insensitive
    next        loop back to the first rule
    skip=N      skip the following N rules
  
  Misc
    -f          file
    -d          directory
  
  Variables encapsulated in: 
    %{VAR_NAME}
  The rule is that mod_rewrite steps through each rewrite rule in turn until one matches or no rules are left.

  Disable hot linking images example:
    RewriteCond %{HTTP_REFERER} !^$  
    RewriteCond %{HTTP_REFERER} !^http://localhost/.*$ [OR,NC]  
    RewriteCond %{HTTP_REFERER} !^http://mysite.com/.*$ [OR,NC]  
    RewriteCond %{HTTP_REFERER} !^http://www.mysite.com/.*$ [OR,NC]  
    RewriteRule .*.(gif|GIF|jpg|JPG)$ http://mysite/images/bad.gif [L,R]
    
  RewriteMap example:
    RewriteMap categories txt:/path/to/file/categoryids.txt    
    RewriteRule ^(.*)$ open.php?categoryid=${categories:$1|0}
    
  Prefix example:
    RewriteCond %{HTTP_HOST} !^mydomain.com [NC]
    RewriteRule ^/?(.*)$ http://mydomain.com/$1 [L,R=301]
  Use similar to guarantee HTTPS!
